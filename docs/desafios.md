# Desafios

Os desafios sao o nucleo do compila.dev. Eles simulam situacoes reais e geram
XP, progresso e visibilidade no feed.

## Como progredir melhor

- Resolva desafios em sequencia na mesma stack.
- Registre o que aprendeu (anotacoes curtas).
- Refaça desafios antigos para fixar conceitos.

## Estrutura tipica de um desafio

- **Descricao**: contexto e requisitos.
- **Starter code**: base inicial para voce evoluir.
- **Testes**: validam regras e casos limite.
- **Feedback**: resultado e recomendacoes.

## Dicas de estudo

- Leia o enunciado completo antes de codar.
- Divida o problema em passos pequenos.
- Teste com exemplos simples antes de enviar.
- Ajuste o codigo com base nos testes que falharam.

## Progresso pessoal

- Use o portfolio para refletir desafios concluídos.
- Publique no feed quando aprender algo relevante.
